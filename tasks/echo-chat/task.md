# Практика: Эхо-чат на сокетах

**Тег коммита:** `echo-chat`
**Баллы:** 6

---

## Описание

Напишите многопользовательский TCP-сервер на сокетах. Пользователи подключаются через telnet, вводят никнейм и общаются в общем чате — каждое сообщение видят все подключённые.

```
$ telnet localhost 9090

Введите никнейм: Alice
[Сервер] Alice вошла в чат. Участников: 1

Alice> Привет!
[Alice] Привет!

[Сервер] Bob вошёл в чат. Участников: 2
[Bob] Привет всем!
```

---

## Требования

### Подключение
- Сервер стартует на порту, переданном первым аргументом (`args[0]`)
- При подключении сервер запрашивает никнейм (первая строка от клиента)
- Никнейм не может быть пустым — сервер запрашивает повторно
- После ввода никнейма всем участникам приходит уведомление: `[Сервер] <Nick> вошёл в чат`

### Сообщения
- Любое сообщение от клиента рассылается всем остальным в формате `[<Nick>] <текст>`
- Отправитель своё сообщение не получает
- Пустые строки игнорируются (не рассылаются)

### Отключение
- При отключении клиента (в том числе внезапном разрыве) всем приходит: `[Сервер] <Nick> покинул чат`
- Сервер продолжает работать при любых отключениях

### Надёжность
- Сервер не падает если клиент отключился во время чтения/записи
- Сервер не падает при пустом вводе, очень длинных строках, спецсимволах
- Несколько клиентов работают одновременно без блокировок друг друга

---

## Что именно нужно сделать

Реализовать многопользовательский чат-сервер на TCP-сокетах:

1. Клиент подключается к серверу.
2. Сервер просит ввести ник.
3. После успешного ввода ника клиент может отправлять сообщения.
4. Сообщения одного участника получают остальные участники чата.
5. Сервер отправляет системные уведомления о входе и выходе участников.
6. Сервер корректно обрабатывает проблемные сценарии (корнер-кейсы).

---

## Критерии оценивания (6 баллов)

### 1 балл — базовый happy path на 2 клиента
- Можно подключиться к серверу.
- Клиент вводит ник.
- Клиент отправляет сообщение.
- Обмен сообщениями корректно работает минимум между 2 клиентами.

### 1 балл — уведомления о входе/выходе
- При входе участника в чат отправляется сообщение: `участник <Nick> вошёл в чат`.
- При выходе участника отправляется сообщение: `участник <Nick> покинул чат`.
- В обоих уведомлениях обязательно используется ник участника.

### 2 балла — масштабирование до 3+ участников
- Чат одновременно работает для 3 и более клиентов.
- Сообщения корректно доставляются всем релевантным участникам.
- Один медленный/проблемный клиент не должен блокировать остальных.

### 2 балла — обработка корнер-кейсов
- Клиент отключился до ввода ника.
- Клиент отправил пустой ник (сервер не принимает, запрашивает повторно).
- Аварийный выход клиента из чата (разрыв соединения, закрытие процесса и т.п.) не приводит к падению сервера.
- После аварийного выхода остальных участников можно продолжать обслуживать.

## Технические ограничения

- Только стандартная библиотека Java (`java.net`, `java.io`, `java.util.concurrent`)
- Класс сервера: `hse.java.EchoServer`
- Точка входа: `public static void main(String[] args)`, порт = `Integer.parseInt(args[0])`
- Никаких фреймворков
