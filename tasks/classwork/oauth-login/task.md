# Практика: OAuth2-авторизация через GitHub и Google

**Тег коммита:** `oauth-login`
**Баллы:** 3

---

## Описание

Добавьте OAuth2-авторизацию через GitHub и/или Google в существующее Spring Boot приложение. Все маршруты защищены — неавторизованный пользователь перенаправляется на страницу входа. При первом входе пользователь сохраняется в базе данных. После авторизации отображаются имя и аватар из OAuth2-профиля.

## Требования

- Страница `/login` содержит кнопки «Войти через GitHub» и «Войти через Google».
- Все маршруты (`/**`) требуют аутентификации; страница `/login` и статические ресурсы доступны без авторизации.
- При каждом входе данные пользователя (имя, аватар) обновляются в базе данных.
- Имя и аватар отображаются в UI; logout перенаправляет на `/login`.
- `client-id` и `client-secret` только из переменных окружения, не из захардкоженных секретов в репозитории.
- Для локальной работы — зарегистрированные OAuth-приложения с callback `http://localhost:8080/login/oauth2/code/{provider}`.
- Spring Boot 3.x, Java 17+, `spring-boot-starter-oauth2-client`, Thymeleaf с поддержкой Spring Security в шаблонах, пользователи в БД через Spring Data JPA. Другие стеки авторизации не допускаются. Минимум один провайдер (GitHub или Google); оба — для максимального балла.

## Проверка

По желанию курса: интеграционные тесты, в которых проверяются **маршруты** (редирект неавторизованного на `/login`, доступ к защищённым страницам после «входа», logout) и обновление данных пользователя; либо приёмка по чеклисту с теми же точками.
