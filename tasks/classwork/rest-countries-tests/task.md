# Практика: Интеграционные тесты для REST Countries API

**Тег коммита:** `rest-countries-tests`
**Баллы:** 4

---

## Описание

Напишите интеграционные тесты к публичному API [REST Countries](https://restcountries.com) (базовый URL: `https://restcountries.com/v3.1`). Проверяете реальный сервис: коды ответа, структуру и содержимое JSON.

Отдельное приложение не требуется — достаточно тестового модуля Maven.

## Требования

| # | Эндпоинт | Проверка |
|---|---------|----------|
| 1 | `GET /all` | 200, список стран не пустой |
| 2 | `GET /name/russia` | 200, есть страна со столицей `"Moscow"` |
| 3 | `GET /alpha/de` | 200, есть страна с `name.common` = `"Germany"` |
| 4 | `GET /name/nonexistentcountryxyz` | 404 |
| 5 | `GET /all` | у каждой страны `population` > 0 |
| 6–7 | на выбор | не менее двух своих сценариев |

Базовый URL задан один раз для всех тестов (константа или настройка клиента в `@BeforeAll`). Проверка `population` охватывает **все** элементы ответа, не только первый. Имена тестов читаемы по смыслу проверки. Заглушки не допускаются.
