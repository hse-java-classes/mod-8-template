# Практика: Чат на Spring Boot + Thymeleaf + STOMP

**Тег коммита:** `chat-stomp`
**Баллы:** 6

---

## Описание

Реализуйте многопользовательский чат в реальном времени с использованием WebSocket и протокола STOMP. Бэкенд — Spring Boot, фронтенд — Thymeleaf-страница с JavaScript.

Сообщения хранятся в памяти, потокобезопасно, без базы данных.

Клиент подключается по SockJS/STOMP, подписывается на топик новых сообщений и отправляет сообщения на сервер. Сервер добавляет временную метку и рассылает сообщение всем подписчикам. При открытии страницы новый клиент сразу получает историю предыдущих сообщений через REST-эндпоинт.

Стек: Spring Boot 3.x, Java 17+, Thymeleaf, WebSocket с SockJS как fallback. Сборка Maven или Gradle.

## API

| Метод | Путь | Тело запроса | Тело ответа | Статус |
|-------|------|--------------|-------------|--------|
| GET | `/` | — | HTML-страница чата | 200 |
| GET | `/history` | — | `[{"sender","text","timestamp"},...]` | 200 |
| STOMP SEND | `/app/send` | `{"sender","text"}` | — | — |
| STOMP SUBSCRIBE | `/topic/messages` | — | `{"sender","text","timestamp"}` | — |

## Требования

- Кнопка отправки заблокирована до установки WebSocket-соединения.
- Каждое входящее сообщение дополняется временной меткой на сервере (формат `HH:mm:ss`).
- При открытии страницы клиент загружает историю через `GET /history` и отображает её до появления новых сообщений.
- Новое сообщение отображается во всех открытых вкладках одновременно без перезагрузки страницы.
- Отправка пустого имени или пустого текста не допускается (проверка на клиенте).

## Проверка

Интеграционные тесты с поднятым приложением: **эндпоинт** `GET /history` — код ответа и JSON по спецификации API; сценарий STOMP — отправка в `/app/send` и получение сообщения в подписке на `/topic/messages` с ожидаемыми полями.
