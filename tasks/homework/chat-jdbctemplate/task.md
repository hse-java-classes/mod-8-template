# Домашняя работа: Чат с хранением сообщений в PostgreSQL через JdbcTemplate

**Тег коммита:** `chat-jdbctemplate`
**Баллы:** 5

---

## Описание

Замените файловое хранилище сообщений (`chat-history`) на **PostgreSQL**. Доступ к БД — через **Spring JDBC** (`JdbcTemplate`). 
Основа — чат на Spring Boot + Thymeleaf + STOMP из лекции.

## API

| Метод | Путь | Тело запроса | Тело ответа | Статус |
|-------|------|--------------|-------------|--------|
| GET | `/` | — | HTML-страница чата | 200 |
| GET | `/history` | — | `[{"sender","text","timestamp"},...]` | 200 |
| STOMP SEND | `/app/send` | `{"sender","text"}` | — | — |
| STOMP SUBSCRIBE | `/topic/messages` | — | `{"sender","text","timestamp"}` | — |

## Требования

- Схема БД создаётся автоматически при старте приложения. Время создания сообщения проставляется при сохранении без участия клиента.
- Новое сообщение сохраняется в БД через `JdbcTemplate`.
- История для клиента: последние N записей по времени (лимит из `chat.history.limit` в `application.properties`); порядок в ответе API — хронологический для отображения.
- Только параметризованные запросы (`?`); конкатенация значений в SQL запрещена.
- Spring Data JPA, встроенные БД вместо PostgreSQL и файловое хранилище сообщений для этой работы не использовать.

## Проверка

Интеграционные тесты с поднятым приложением и **реальной тестовой PostgreSQL**. Проверьте **эндпоинт** `GET /history`: код ответа и JSON в формате API. После отправки сообщения по STOMP убедитесь, что новое сообщение доступно через повторный `GET /history` или отражено в БД в тесте. Недостаточно проверять только мок `DataSource` / `JdbcTemplate` без реальной БД.
