# Домашняя работа: История сообщений чата на диске

**Тег коммита:** `chat-history`
**Баллы:** 5

---

## Описание

Расширьте STOMP-чат из практики: сообщения сохраняются на диск в JSON-файл и переживают перезапуск сервера. При старте приложения история читается с диска; новый клиент получает последние N сообщений.

## Требования

- Каждое новое сообщение попадает в файл (путь по умолчанию `data/messages.json`, настраивается в `application.properties`).
- Файл — JSON-массив объектов с полями `sender`, `content`, `timestamp` (ISO-8601, время задаёт сервер).
- Нет файла или пустой файл — старт без ошибок, пустая история.
- Новому клиенту отдаются последние N сообщений (по умолчанию 50, лимит `chat.history.limit`).
- Запись и обновление памяти при конкурентных отправках — потокобезопасны.
- Путь к данным в `.gitignore` (чтобы не коммитить реальные сообщения).
- Формат файла — JSON; ручной разбор строк не допускается.
- Базы данных не используются; дополнительные зависимости сверх `spring-boot-starter-web` / `spring-boot-starter-websocket` не подключать.
