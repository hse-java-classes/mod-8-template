# Домашняя работа: Telnet-чат с комнатами

**Тег коммита:** `telnet-chat`
**Баллы:** 7

---

## Описание

Напишите многопользовательский TCP-сервер с поддержкой чат-комнат. Пользователь подключается через telnet, вводит никнейм и попадает в главное меню, из которого может создавать комнаты, просматривать список существующих и заходить в них для общения.

```
$ telnet localhost 9090

Введите никнейм: Alice

=== Главное меню ===
1. Создать комнату
2. Список комнат
3. Подключиться к комнате
0. Выйти

> 1
Введите название комнаты: java-talks
[Сервер] Комната "java-talks" создана.

=== Главное меню ===
> 2
Комнаты:
  java-talks (1 участник)

=== Главное меню ===
> 3
Введите название комнаты: java-talks
[Сервер] Alice вошла в комнату java-talks.

Alice> Привет!
[Alice] Привет!
[Bob] Привет, Alice!

/exit
[Сервер] Alice покинула комнату.

=== Главное меню ===
```

---

## Что нужно реализовать

Реализуйте TCP-сервер чата с комнатами:

1. Пользователь подключается к серверу и вводит ник.
2. После входа пользователь работает через главное меню.
3. Через меню можно создать комнату, посмотреть список комнат и зайти в комнату.
4. Внутри комнаты участники обмениваются сообщениями.
5. Поддерживаются системные события входа/выхода участников.
6. Поддерживается удаление комнаты владельцем.
7. Сервер устойчив к некорректным действиям и обрывам соединения.

---

## Подробные требования

### Подключение и меню
- Сервер стартует на порту из `args[0]`
- При подключении сервер запрашивает никнейм (непустой)
- После ввода никнейма показывается главное меню
- Меню показывается повторно после каждого действия (кроме нахождения в комнате)
- Неизвестная команда — сообщение об ошибке и повторный показ меню

### Создание комнаты (`1`)
- Запрашивает название комнаты
- Название не может быть пустым или совпадать с уже существующим
- Создавшего считается владельцем комнаты
- После создания пользователь остаётся в главном меню (не заходит в комнату автоматически)

### Список комнат (`2`)
- Выводит все существующие комнаты в формате: `  <название> (<N> участников)`
- Если комнат нет: `Комнат пока нет.`
- Участниками считаются только те, кто сейчас находится внутри комнаты

### Подключение к комнате (`3`)
- Запрашивает название комнаты
- Если комната не существует — сообщение об ошибке, возврат в меню
- При входе все участники комнаты получают: `[Сервер] <Nick> вошёл в комнату`
- Сообщения внутри комнаты рассылаются только участникам этой комнаты в формате `[<Nick>] <текст>`
- Пустые строки игнорируются
- Команда `/exit` — выход из комнаты обратно в главное меню, остальные получают: `[Сервер] <Nick> покинул комнату`

### Удаление комнаты
- Команда `/delete` доступна только внутри комнаты и только владельцу
- Не-владелец получает: `[Сервер] Только создатель комнаты может её удалить.`
- При удалении все участники получают: `[Сервер] Комната "<название>" удалена.` и возвращаются в главное меню
- Удалённую комнату можно создать повторно с тем же именем (имя освобождается)

### Отключение
- При разрыве соединения (в том числе внезапном) пользователь убирается из комнаты/сервера
- Если отключился владелец комнаты, которая не удалена — комната остаётся, владелец не назначается
- Остальные участники получают уведомление об уходе

### Надёжность
- Сервер не падает ни при каких действиях клиента
- Все операции над общим состоянием (список комнат, участники) потокобезопасны
- Несколько сотен одновременных подключений не вызывают деградации

### Стек
- Только стандартная библиотека Java (`java.net`, `java.io`, `java.util.concurrent`); без фреймворков.
- Класс сервера: `hse.java.TelnetChatServer`; точка входа `public static void main(String[] args)`.
