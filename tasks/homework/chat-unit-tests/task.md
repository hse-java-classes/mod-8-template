# Домашняя работа: Тесты для чата с историей на диске

**Тег коммита:** `chat-unit-tests`
**Баллы:** 6

---

## Описание

Покройте автотестами поведение чата с историей на диске (`chat-history`): REST, доставка через STOMP и согласованность с файлом истории. Базовый подход — **интеграционные** тесты с поднятием Spring-приложения; изоляция за счёт тестового профиля и отдельного пути к файлу (каталог под `target/`), без использования production-пути.

## Что должны подтвердить тесты

- **REST:** `GET /history` — успешный ответ и тело в ожидаемом формате; если перед стартом положить валидный JSON по тестовому пути, после поднятия контекста история непустая.
- **STOMP:** клиент к работающему приложению подписывается на топик сообщений, отправляет payload по контракту `chat-history` и получает рассылку; проверяется содержимое (отправитель, текст, метка времени — по спецификации задания).
- **Файл:** после сценария с новым сообщением тестовый файл на диске отражает сохранённые сообщения (корректный JSON-массив, появление записи).

Нужно не менее **6** содержательных проверок (отдельные `@Test` или параметризация). Заглушки вроде `assertTrue(true)` запрещены. Узкие unit-тесты без контекста допустимы только как дополнение, не вместо проверки REST/STOMP и файла.
